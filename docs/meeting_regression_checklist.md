# Meeting Regression Checklist

## 계정 구성
- 관리자 1명 + 일반 멤버 1명으로 교차 점검

## A. 상태 전이
1. 세션 지원 마감 전/후 버튼 노출이 의도대로 바뀌는지
2. 매칭 실행 -> 조율중 진입 확인
3. 공유 후 예약 진입(`schedule_booking_start`) 시 booking 플래그 반영 확인
4. 최종 공유 시 version 증가 및 공개 상태 반영 확인
5. 최종 확정 후 변경 차단 확인

## B. 세션 배정/지원
1. 일반 멤버 지원/취소 AJAX 동작
2. 배정된 세션에서 지원 취소 차단 확인
3. 관리자 배정/해제 시 UI와 DB 상태 일치 확인

## C. 최종 확인(version)
1. `schedule_version` 상승 후 기존 ack가 무효 처리되는지
2. 현재 버전 ack만 집계되는지

## D. meeting_detail UI
1. 관리자/일반인 레이아웃 분기가 정상인지
2. 세션 지원자 펼침/접기 상태별 세션 정렬(가운데/상단) 정상인지
3. hover, 숫자열, 버튼줄 정렬이 깨지지 않는지
4. 세션 지원자 수 뱃지가 정적/동적 업데이트 모두 동일 스타일인지
5. 페이지네이션 30곡 기준에서 정렬/필터/AJAX 동작이 정상인지

## E. 데이터 정책
1. 임시 합주실이 실제 합주실 데이터 의미를 훼손하지 않는지
2. 최종 확정 상태에서 조작 API가 차단되는지

## F. 매칭 결과 일치성 (신규 필수)
1. 주차별 `직접 배치 필요` 카드와 상단 `매칭 성공 n곡` 수치가 동일 기준으로 변하는지
2. 전체 `배치 불가 목록`과 주차별 카드가 동일 곡/주차를 가리키는지
3. 새로고침/임시저장 복원(`load_saved`) 후에도 1,2 항목이 유지되는지

## F-1. 예약 단계 버튼 상태 (신규 필수)
1. `예약 임시 저장`은 카드 이동/리사이즈뿐 아니라 예약 확정/취소 클릭에도 즉시 활성화되는지
2. `저장 후 공유`는 공유 직후 비활성화되고, 이후 변경이 생기면 즉시 다시 활성화되는지
3. 예약 진입 버튼은 공유 이후 변경 시 비활성화되고, 안내 문구가 버튼 바로 위에서 함께 바뀌는지

## G. 공간 부족 사유 정책 (신규 필수)
1. 같은 미팅 내 점유로 인한 `사용 중` 사유가 노출되지 않는지
2. 외부 미팅 사유만 `[미팅 이름] - 곡명` 형식으로 표시되는지
3. 외부 미팅 사유는 주황 계열로 표시되는지
4. 합주실 설정 기반 블록은 `예약 불가`로 유지되는지

## J. 추가합주 RoomBlock 일관성 (신규 필수)
> `extra_practice_views.py`에서 `ExtraPracticeSchedule` + `RoomBlock` 생성이 `transaction.atomic()` 없이 수행됨.
> 동시 요청 경합 시 한쪽만 성공할 수 있으므로 DB 상태 일관성 모니터링이 필요.

1. 추가합주 저장 후 DB에 `ExtraPracticeSchedule` 레코드와 연관 `RoomBlock` 레코드가 동시에 생성되는지 확인
2. 추가합주 삭제 후 연관 `RoomBlock`도 함께 삭제되는지 확인
3. 동일 시간대에 두 요청이 동시 도달할 때 중복 충돌 409 응답이 반환되는지 확인 (트랜잭션 부재 위험 모니터링용)
4. 저장 성공 응답에 `schedule_id` + `block_id` 모두 포함되는지 확인 (한쪽만 성공한 경우 식별용)

## H. 검증 운영 원칙
1. 기본은 `./.venv/bin/python manage.py check` + 기능 단위 스모크
2. `manage.py test`는 상시 루틴으로 매번 실행하지 않음

## H-1. 룸슬롯 단일 모드 (신규 필수)
1. 관리자 조율 화면에서 `일반/룸 보기` 토글이 노출되지 않는지
2. 보드가 roomslot 기준으로만 렌더되는지
3. `빈 합주실 가리기` 계열 버튼이 재등장하지 않는지

## I. 타일 크기 조절/부족시간 연동 (신규 필수)
1. 같은 주차 내에서만 드래그 이동이 되는지(다른 주차로는 드롭 불가)
2. 배치된 곡 카드의 상단/하단 엣지에서 30분 단위 리사이즈가 되는지
3. 리사이즈 직후 카드 좌측에 `초과`(파란색) / `부족`(주황색) 배지가 즉시 갱신되는지
4. `강제` 배지(우측)와 `초과/부족` 배지(좌측)가 겹치지 않는지
5. 부족이 생기면 같은 주차 `직접 배치 필요` 카드의 `부족한 합주시간`이 실시간 증가하는지
6. 부족이 줄면 `직접 배치 필요` 카드의 `부족한 합주시간`이 실시간 감소하는지
7. 부족이 0이면 해당 `직접 배치 필요` 카드가 사용 처리(또는 숨김)되는지
8. `직접 배치 필요` 카드를 드래그 배치할 때도 부족시간이 실시간 반영되는지
9. 분할(롱프레스) 후 조각 리사이즈 시에도 총합 기준으로 부족시간/배지가 일치하는지
10. `임시 저장` 후 `load_saved` 복원 시 리사이즈 결과(시작/종료/duration)와 배지 상태가 유지되는지
