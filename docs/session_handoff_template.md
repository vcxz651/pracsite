# Session Handoff Template

이 문서는 **현재 작업 세션 종료 시 다음 AI 작업자에게 넘길 handoff 형식**이다.
프로젝트 SSOT를 대체하지 않고, "이번 세션에서 무엇을 했는지"만 남긴다.

원칙:
- 작업을 마칠 때는 이 형식으로 정리한다.
- 같은 세션 요약을 여러 문서에 흩어 쓰지 않는다.
- 기능 기준은 기존 SSOT 문서에 반영하고, 이 문서는 세션 메타데이터만 남긴다.

---

## 사용 규칙

1. 작업 종료 직전에 이 템플릿 형식으로 handoff를 작성한다.
2. 실제 기준이 바뀐 내용은 관련 SSOT 문서에도 함께 반영한다.
3. handoff에는 "이번 세션의 변경/미변경/남은 리스크"만 적는다.
4. 추측하지 말고, 실제로 실행한 것만 적는다.

권장 위치:
- 대화 종료 답변에 이 형식을 따라 정리
- 필요 시 별도 임시 handoff 문서에 저장 후 다음 세션에서 삭제/갱신

---

## 템플릿

### 1. 작업 범주
- 예: 매칭/예약 동작 수정
- 예: `meeting_detail` UI 안정화
- 예: 데모 시나리오 문구 수정

### 2. 이번 세션에서 실제 수정한 파일
- 파일 경로만 적지 말고, 무엇을 바꿨는지 한 줄씩 같이 적는다.
- 예:
  - `pracapp/templates/pracapp/match_result.html` — 저장/예약 진입 중복 클릭 방어
  - `pracapp/views/matching_views.py` — 최종 확정 동시 요청 방어
  - `docs/ai_working_rules.md` — 중복 요청 방지 규칙 문서화

### 3. 바꾼 것
- 사용자에게 보이는 변경
- 내부 안정성 변경
- 문서 기준 변경

짧게:
- 무엇이 이제 가능/안전해졌는가
- 어떤 버그/리스크를 줄였는가

### 4. 의도적으로 안 바꾼 것
- 핵심 정책
- 상태 전이 규칙
- 권한 규칙
- DB 스키마

짧게:
- 무엇은 그대로 유지했는지 적는다.

### 5. 실행한 검증
- 실제로 실행한 명령만 적는다.
- 예:
  - `./.venv/bin/python manage.py check`
  - 수동으로 버튼 연타 스모크 확인

### 6. 실행하지 않은 검증
- 안 돌린 테스트/미확인 동작을 적는다.
- 예:
  - `./.venv/bin/python manage.py test pracapp.tests` 미실행
  - 브라우저 실클릭 미확인

### 7. 남은 리스크 / 다음 AI가 먼저 볼 것
- 아직 손대지 않은 위험 구간
- 다음 작업자가 먼저 볼 파일/문서
- 지금 수정과 인접해서 추가로 확인할 포인트

예:
- `meeting_detail.html`의 다른 전역 이벤트 위임 충돌 가능성은 추가 확인 필요
- 다음 작업자는 `docs/large_files_overview.md`의 해당 파일 규칙부터 보고 시작

### 8. 다음 AI 첫 순서
1. 먼저 읽을 문서
2. 먼저 볼 파일
3. 바로 하면 안 되는 추정

예:
1. `docs/meeting_handover.md`, `docs/meeting_regression_checklist.md`
2. `pracapp/templates/pracapp/match_result.html`, `pracapp/views/matching_views.py`
3. 클라이언트 버튼 상태만 바꾸면 서버 검증도 따라간다고 추정하지 말 것

---

## 최소 품질 기준

- "무엇을 했다"보다 "무엇을 안 바꿨는지"가 같이 있어야 한다.
- 검증은 실행/미실행을 반드시 분리한다.
- 다음 AI가 첫 5분 안에 어디서 시작할지 결정할 수 있어야 한다.
